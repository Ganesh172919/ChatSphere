# Persistence Models Import Export And Project Context

- This file explains how the backend stores AI artifacts and how import, export, and project context extend the AI system.
- The persistence model matters because explainability, auditing, and UI behavior all depend on what is saved alongside AI responses.
- Use this file when changing schemas, adding audit metadata, or expanding import/export compatibility.

### `backend/models/Conversation.js`

- Responsibility: Stores solo chat history and assistant runtime metadata.
- Assistant messages can store memoryRefs, attachment metadata, modelId, provider, requestedModelId, processingMs, token counts, autoMode, autoComplexity, and fallbackUsed.
- Conversations can also store projectId, projectName, sourceType, sourceLabel, importFingerprint, and importSessionId.
- The schema is indexed for per-user recent conversation queries and per-user project queries.

### `backend/models/Message.js`

- Responsibility: Stores room messages and room AI replies.
- Room messages persist reply previews, reaction maps, delivery status, pin state, edit history, delete state, attachment metadata, memoryRefs, modelId, and provider.
- AI room replies are normal messages with isAI true and userId ai.
- The schema carries a text index for search and room and date-oriented indexes for recent message retrieval.

### `backend/models/Room.js`

- Responsibility: Stores room-level AI prompt history in aiHistory.
- New rooms seed aiHistory with an initial prompt exchange through buildInitialRoomHistory.
- The seed prompt references the room name so the assistant starts with room-specific framing.
- Live trigger_ai calls append user and model turns to this array and later trim it.

### `backend/models/Project.js`

- Responsibility: Stores solo-chat project context and project files.
- Projects can store instructions, long-form context, tags, suggested prompts, and attached files.
- Project files reuse the same attachment shape as uploads used in direct chat flows.
- The AI gateway can turn project files into prompt text or image payloads through the same attachment builder used elsewhere.

### `backend/services/importExport.js`

- Responsibility: Parses external history formats and emits portability formats.
- Supports ChatGPT-style JSON parsing, Claude JSON or transcript parsing, and generic markdown or text parsing.
- Uses SHA-1 fingerprints to avoid duplicate import sessions and duplicate conversations.
- Creates Conversation records for imported histories.
- Creates memory entries from imported user text using upsertMemoryEntries.
- Refreshes conversation insights for imported conversations.
- Exports normalized JSON, markdown, and adapter-shaped JSON.

### `backend/models/ImportSession.js`

- Responsibility: Tracks preview and import sessions for external histories.
- Stores sourceType, sourceName, fingerprint, preview counts, errors, and arrays of imported conversation and memory ids.
- Keeps imports idempotent per user and content fingerprint.

## How Project Context Reaches The Model

- The solo chat route resolves projectId from the request body or from the existing conversation project.
- The route loads the project only for the current user.
- The gateway calls buildProjectContext(project) when a project is present.
- Project context includes project name, description, instructions, general context, tags, suggested prompts, and up to six attached project files.
- Each included project file is transformed through buildAttachmentPayload and truncated before insertion into the prompt.
- Room AI currently has no equivalent projectId input path.

## How Export Uses AI Metadata

- Normalized exports include the raw persisted conversation, insight, and memory documents.
- Markdown exports summarize memories and then print conversation turns in order.
- Adapter exports expose a provider label, memories, conversations, and matched insights in a generic downstream shape.
- Room exports include AI messages, edited state, attachments, and memoryRefs where available.

## Schema Alignment Rules

- If you add new assistant metadata fields to Conversation.messages, update the chat route, conversation detail route, shared frontend types, and any export path that should surface the new data.
- If you add new room AI metadata fields to Message, update room formatting helpers, room detail routes, socket event serializers, search, and export payloads as needed.
- If you change the project file shape, keep it compatible with validateAttachmentPayload and buildAttachmentPayload.
- If you alter import parsing, ensure preview and import modes stay aligned so users see realistic previews before commit.
