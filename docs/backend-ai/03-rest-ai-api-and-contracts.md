# REST AI API And Contracts

- This file documents the AI-relevant REST surface that the frontend and future integrations consume today.
- The focus is on actual contracts implemented in the backend, including gating rules, metadata fields, and failure behavior.
- Routes that are not generation endpoints but still shape the AI experience are included here when they affect models, memories, insights, or prompt selection.

### `backend/routes/ai.js`

- Responsibility: Exposes utility AI routes for models, smart replies, sentiment, and grammar.
- GET /api/ai/models refreshes remote catalogs, returns visible deployment models, and prepends a client-facing auto option.
- POST /api/ai/smart-replies requires auth, aiLimiter, and aiQuotaMiddleware before generating suggestions.
- POST /api/ai/sentiment requires auth, aiLimiter, and aiQuotaMiddleware before analyzing text.
- POST /api/ai/grammar requires auth, aiLimiter, and aiQuotaMiddleware before correcting text.
- Each utility route checks user feature toggles through loadAiPreferences before generating.
- Each utility route uses deterministic fallbacks when the gateway cannot return structured JSON safely.

### `backend/routes/chat.js`

- Responsibility: Implements POST /api/chat for solo AI conversations.
- Requires auth and aiQuotaMiddleware.
- Accepts message, optional conversationId, optional history, optional modelId, optional attachment, and optional projectId.
- Returns conversationId, content, timestamp, optional memoryRefs, optional insight, modelId, provider, requestedModelId, processingMs, token counts, autoMode, autoComplexity, and fallbackUsed.
- Returns 429 for provider rate-limit conditions and 503 for broader provider unavailability when the gateway throws an AI-specific error.
- Returns 404 if a referenced project does not belong to the user.
- Rejects cross-project conversation usage if the provided projectId does not match the existing conversation project.

### `backend/routes/conversations.js`

- Responsibility: Exposes conversation history and AI-generated insight endpoints.
- GET /api/conversations returns summaries with message counts and project linkage.
- GET /api/conversations/:id returns persisted assistant metadata including provider and token counts.
- GET /api/conversations/:id/insights returns an existing or newly generated conversation insight.
- POST /api/conversations/:id/actions/summarize refreshes insight and returns summary plus full insight payload.
- POST /api/conversations/:id/actions/extract-tasks refreshes insight and returns action items plus full insight payload.
- POST /api/conversations/:id/actions/extract-decisions refreshes insight and returns decisions plus full insight payload.

### `backend/routes/rooms.js`

- Responsibility: Exposes room insight REST endpoints that complement the socket room AI flow.
- GET /api/rooms/:id returns room messages plus the current room insight.
- GET /api/rooms/:id/insights returns the existing or newly generated room insight for a member.
- POST /api/rooms/:id/actions/summarize refreshes room insight and returns summary plus the full insight object.
- POST /api/rooms/:id/actions/extract-tasks refreshes room insight and returns action items plus the full insight object.
- POST /api/rooms/:id/actions/extract-decisions refreshes room insight and returns decisions plus the full insight object.
- These routes do not trigger room AI replies; they only generate structured analysis output.

### `backend/routes/memory.js`

- Responsibility: Exposes AI memory CRUD plus import/export helper endpoints.
- GET /api/memory returns memory entries filtered by search, pinned state, and limit.
- PUT /api/memory/:id updates summary, details, tags, pin state, and scoring fields.
- DELETE /api/memory/:id removes a memory entry for the current user.
- POST /api/memory/import supports preview and import modes for external AI history content.
- GET /api/memory/export returns normalized JSON, adapter JSON, or markdown exports.

### `backend/routes/settings.js`

- Responsibility: Controls user AI feature toggles.
- GET /api/settings returns the persisted settings object or default settings.
- PUT /api/settings updates settings.aiFeatures.smartReplies, settings.aiFeatures.sentimentAnalysis, and settings.aiFeatures.grammarCheck when provided.
- These flags are consumed by backend/routes/ai.js as backend-enforced guards.

### `backend/routes/admin.js`

- Responsibility: Controls prompt templates that influence backend AI behavior.
- GET /api/admin/prompts returns merged default and database-backed prompt templates.
- PUT /api/admin/prompts/:key writes or updates prompt content, version, and description.
- Admin prompt changes affect future AI calls because runtime prompt resolution checks the database first.

## Core Request Contracts

- GET /api/ai/models returns models, defaultModelId, optional hasConfiguredModels, and optional emptyStateMessage.
- POST /api/ai/smart-replies expects a non-empty messages array and optional context and modelId.
- POST /api/ai/sentiment expects a text string and optional modelId.
- POST /api/ai/grammar expects a text string of at least 3 characters and optional modelId.
- POST /api/chat expects a non-empty message string and optionally accepts conversationId, history, modelId, attachment, and projectId.
- Conversation and room insight action routes accept an optional body with modelId to influence the summarization model selection.
- Memory import expects content, filename, and optional mode where mode is preview or import.
- Project routes accept validated file metadata arrays so project files can later be injected into solo AI prompts.

## Important Response Metadata

- Assistant responses from /api/chat persist and return the selected concrete modelId.
- Assistant responses from /api/chat persist and return the selected concrete provider.
- Assistant responses from /api/chat persist and return requestedModelId so the UI can compare requested versus actual routing.
- Assistant responses from /api/chat persist and return processingMs for runtime diagnostics.
- Assistant responses from /api/chat persist and return promptTokens, completionTokens, and totalTokens when the provider returns usage.
- Assistant responses from /api/chat persist and return autoMode, autoComplexity, and fallbackUsed so the UI can explain routing behavior.
- Conversation detail routes return the same metadata back out of persistence so history stays explainable.
- Room detail routes return AI Message entries with memoryRefs, modelId, and provider for room AI messages.

## Failure Contracts

- Utility routes return 403 when the feature is disabled in user settings.
- Utility routes return 400 when required text or message arrays are missing.
- Utility routes can return 429 when route limits or in-memory AI quota is exhausted.
- Solo chat returns 429 when the gateway surfaces a provider rate-limit condition.
- Solo chat returns 503 when the gateway surfaces a temporary provider failure or model unavailability condition.
- Memory routes return 404 when a memory entry or import target is missing for the current user.
- Conversation routes return 404 when the requested conversation is not owned by the current user.
- Admin prompt routes return 400 when the key or content is empty and 500 on persistence failures.
