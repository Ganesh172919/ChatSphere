# Database Schema

ChatSphere uses MongoDB as its primary datastore, interacted with via Mongoose ODM. Below are the definitions of all 7 collections.

## Collections

### 1. Users (`users` collection)
Stores all user account profiles, authentication credentials, preferences, and moderation state.

| Field | Type | Attributes | Description |
| :--- | :--- | :--- | :--- |
| `_id` | ObjectId | Primary Key | MongoDB auto-generated ID. |
| `username` | String | required, unique, min:3, max:30, lowercase | Unique handle for the user. |
| `email` | String | required, unique, lowercase | User's email address. |
| `passwordHash` | String | default: null | Bcrypt hash of password (null if Google Auth). |
| `googleId` | String | default: null, sparse | Google OAuth profile ID. |
| `avatar` | String | default: null | URL or base64 data URL for profile image. |
| `displayName` | String | default: null | Formatted display name. |
| `bio` | String | max:200, default: "" | User's short biography. |
| `authProvider` | String | enum: ['local', 'google'] | Authentication method used to create account. |
| `onlineStatus` | String | enum: ['online', 'away', 'offline'], default: 'offline' | Current presence status. |
| `lastSeen` | Date | default: Date.now | Last activity timestamp. |
| `settings.theme.mode` | String | enum: ['dark', 'light', 'system'], default: 'dark' | UI theme preference. |
| `settings.theme.customTheme` | String | default: 'default' | Custom theme variant. |
| `settings.accentColor` | String | default: '#A855F7' | UI accent color (hex). |
| `settings.notifications.sound` | Boolean | default: true | Enable notification sounds. |
| `settings.notifications.desktop` | Boolean | default: true | Enable desktop notifications. |
| `settings.notifications.mentions` | Boolean | default: true | Enable mention alerts. |
| `settings.notifications.replies` | Boolean | default: true | Enable reply alerts. |
| `settings.aiFeatures.smartReplies` | Boolean | default: true | Enable AI smart reply suggestions. |
| `settings.aiFeatures.sentimentAnalysis` | Boolean | default: false | Enable sentiment badges. |
| `settings.aiFeatures.grammarCheck` | Boolean | default: false | Enable grammar checking. |
| `blockedUsers` | Array of ObjectId | Ref: `User` | IDs of users blocked by this user. |
| `isAdmin` | Boolean | default: false | Platform admin flag. |
| `createdAt` | Date | auto | Document creation timestamp. |
| `updatedAt` | Date | auto | Document last update timestamp. |

*Methods:*
- `comparePassword(candidate)` — Async bcrypt comparison
- `toSafeObject()` — Returns user data without sensitive fields (passwordHash, googleId, etc.)
- Pre-save hook: Auto-hashes `passwordHash` with 12 salt rounds when modified

### 2. Rooms (`rooms` collection)
Stores metadata for group chat rooms, member lists with roles, and shared persistent AI conversation history.

| Field | Type | Attributes | Description |
| :--- | :--- | :--- | :--- |
| `_id` | ObjectId | Primary Key | MongoDB auto-generated ID. |
| `name` | String | required, max:50 | Display name of the room. |
| `description` | String | max:500 | Optional details about the room's purpose. |
| `tags` | Array of Strings | lowercase | Categorization labels. |
| `maxUsers` | Number | min:2, max:100, default:20 | Capacity limit for the room. |
| `creatorId` | ObjectId | required, Ref: `User` | ID of the user who created the room. |
| `members` | Array of Objects | — | Room membership list with roles. |
| `members[].userId` | ObjectId | Ref: `User` | Member's user ID. |
| `members[].role` | String | enum: ['admin', 'moderator', 'member'] | Member's role in the room. |
| `members[].joinedAt` | Date | default: Date.now | When the member joined. |
| `pinnedMessages` | Array of ObjectId | Ref: `Message` | IDs of pinned messages. |
| `aiHistory` | Array of Objects | — | Contextual history for Gemini AI in this room. |
| `aiHistory[].role` | String | enum: ['user', 'model'] | Speaker identity in the AI context. |
| `aiHistory[].parts[].text`| String | — | Actual text content sent to or returned by the AI. |
| `createdAt` | Date | auto | Timestamp of room creation. |
| `updatedAt` | Date | auto | Timestamp of last room update. |

*Indexes:*
- `creatorId` (Ascending)
- `tags` (Ascending)

---

### 3. Messages (`messages` collection)
Stores individual chat messages sent within Group Rooms, including reactions, replies, pins, and read receipts.

| Field | Type | Attributes | Description |
| :--- | :--- | :--- | :--- |
| `_id` | ObjectId | Primary Key | MongoDB auto-generated ID. |
| `roomId` | ObjectId | required, index, Ref: `Room` | ID of the room this message belongs to. |
| `userId` | String | required | ID of the sender (or 'system'/'ai'). |
| `username` | String | required | Sender's display name. |
| `content` | String | required | Primary text content of the message. |
| `isAI` | Boolean | default: false | Flag indicating if the message was generated by Gemini. |
| `triggeredBy` | String | default: null | If `isAI=true`, the username who invoked the AI. |
| `replyTo.id` | String | default: null | ID of the parent message this replies to. |
| `replyTo.username` | String | default: null | Username of the parent message author. |
| `replyTo.content` | String | default: null | Snippet of the parent message content (max 100 chars). |
| `reactions` | Map of [String] | default: {} | Key-value mapping of Emoji → Array of User IDs. |
| `status` | String | enum: ['sent', 'delivered', 'read'], default: 'sent' | Message delivery status. |
| `readBy` | Array of Objects | — | Users who have read the message. |
| `readBy[].userId` | String | — | ID of user who read the message. |
| `readBy[].readAt` | Date | — | When the user read the message. |
| `isPinned` | Boolean | default: false | Whether the message is pinned. |
| `pinnedBy` | String | default: null | Username who pinned the message. |
| `pinnedAt` | Date | default: null | When the message was pinned. |
| `createdAt` | Date | auto | Message dispatch timestamp. |
| `updatedAt` | Date | auto | Message edit/update timestamp. |

*Indexes:*
- Compound: `{ roomId: 1, createdAt: -1 }` (Optimizes fetching recent messages for a room)
- Text: `{ content: "text" }` (Enables full-text search)

---

### 4. Conversations (`conversations` collection)
Stores entire histories of private 1-on-1 chats a user has with the AI.

| Field | Type | Attributes | Description |
| :--- | :--- | :--- | :--- |
| `_id` | ObjectId | Primary Key | MongoDB auto-generated ID. |
| `userId` | ObjectId | required, index, Ref: `User` | The owner of the conversation. |
| `title` | String | required, max:100 | Generated title for the chat thread. |
| `messages` | Array of Objects | — | The chronological sequence of messages. |
| `messages[].role` | String | enum: ['user', 'assistant'] | Who sent the message. |
| `messages[].content`| String | required | Markdown body of the message. |
| `messages[].timestamp`| Date | default: Date.now | When this specific message was recorded. |
| `createdAt` | Date | auto | Conversation creation time. |
| `updatedAt` | Date | auto | Last time a message was appended. |

*Indexes:*
- Compound: `{ userId: 1, updatedAt: -1 }` (Optimizes fetching a user's recent chats)

---

### 5. Refresh Tokens (`refreshtokens` collection)
Manages long-lived authentication tokens mapped to specific users.

| Field | Type | Attributes | Description |
| :--- | :--- | :--- | :--- |
| `_id` | ObjectId | Primary Key | MongoDB auto-generated ID. |
| `token` | String | required, unique, index | The secure random refresh token string. |
| `userId` | ObjectId | required, index, Ref: `User`| The user to whom this token belongs. |
| `expiresAt` | Date | required | Expiration timestamp. |
| `createdAt` | Date | auto | Issuance timestamp. |
| `updatedAt` | Date | auto | Last update timestamp. |

*Indexes:*
- TTL Index: `{ expiresAt: 1 }` with `expireAfterSeconds: 0`. MongoDB automatically deletes documents when `expiresAt` is reached.

---

### 6. Polls (`polls` collection)
Stores interactive polls created within group rooms.

| Field | Type | Attributes | Description |
| :--- | :--- | :--- | :--- |
| `_id` | ObjectId | Primary Key | MongoDB auto-generated ID. |
| `roomId` | ObjectId | required, Ref: `Room` | The room this poll belongs to. |
| `creatorId` | ObjectId | required, Ref: `User` | The user who created the poll. |
| `creatorUsername` | String | required | Display name of the creator. |
| `question` | String | required, trim, max:500 | The poll question text. |
| `options` | Array of Objects | — | Poll answer options. |
| `options[].text` | String | required, trim, max:200 | Option text. |
| `options[].votes` | Array of ObjectId | Ref: `User` | User IDs who voted for this option. |
| `allowMultipleVotes` | Boolean | default: false | Whether users can vote on multiple options. |
| `isAnonymous` | Boolean | default: false | Whether voter identities are hidden. |
| `expiresAt` | Date | default: null | Auto-close timestamp (null = no expiry). |
| `isClosed` | Boolean | default: false | Whether the poll has been manually closed. |
| `createdAt` | Date | auto | Poll creation timestamp. |
| `updatedAt` | Date | auto | Last update timestamp. |

*Indexes:*
- Compound: `{ roomId: 1, createdAt: -1 }` (Optimizes listing polls by room)

*Virtuals:*
- `totalVotes` — Computed sum of all option vote counts

*Methods:*
- `hasUserVoted(userId)` — Returns whether the user has voted on any option
- `isExpired()` — Returns whether the poll's expiry time has passed

### 7. Reports (`reports` collection)
Stores user and content moderation reports.

| Field | Type | Attributes | Description |
| :--- | :--- | :--- | :--- |
| `_id` | ObjectId | Primary Key | MongoDB auto-generated ID. |
| `reporterId` | ObjectId | required, Ref: `User` | The user who submitted the report. |
| `targetType` | String | required, enum: ['user', 'message'] | What is being reported. |
| `targetId` | ObjectId | required | ID of the reported user or message. |
| `roomId` | ObjectId | default: null, Ref: `Room` | Room context (if applicable). |
| `reason` | String | required, enum | Reason for the report. |
| `description` | String | max:1000, default: "" | Additional details from the reporter. |
| `status` | String | enum: ['pending', 'reviewed', 'action_taken', 'dismissed'], default: 'pending' | Current report status. |
| `reviewedBy` | ObjectId | default: null, Ref: `User` | Admin who reviewed the report. |
| `reviewNote` | String | default: "" | Admin's review notes. |
| `createdAt` | Date | auto | Report submission timestamp. |
| `updatedAt` | Date | auto | Last update timestamp. |

*Valid reasons:* `spam`, `harassment`, `hate_speech`, `inappropriate_content`, `impersonation`, `other`

*Indexes:*
- `{ reporterId: 1 }` — Lookup reports by reporter
- `{ status: 1, createdAt: -1 }` — Optimizes admin report listing
- `{ targetType: 1, targetId: 1 }` — Prevents duplicate reports on same target
