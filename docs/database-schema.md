# Database Schema

ChatSphere uses MongoDB as its primary datastore, interacted with via Mongoose ODM. Below are the definitions of the main collections.

## Collections

### 1. Users (`users` collection)
Stores all user account profiles and authentication credentials.

| Field | Type | Attributes | Description |
| :--- | :--- | :--- | :--- |
| `_id` | ObjectId | Primary Key | MongoDB auto-generated ID. |
| `username` | String | required, unique, mix:3, max:30 | Unique handle for the user. |
| `email` | String | required, unique | User's email address. |
| `passwordHash` | String | default: null | Bcrypt hash of password (null if Google Auth). |
| `googleId` | String | default: null, sparse | Google OAuth profile ID. |
| `avatar` | String | default: null | URL to user's profile image. |
| `displayName` | String | default: null | Formatted display name. |
| `authProvider` | String | enum: ['local', 'google'] | Authentication method used to create account. |
| `createdAt` | Date | auto | Document creation timestamp. |
| `updatedAt` | Date | auto | Document last update timestamp. |

### 2. Rooms (`rooms` collection)
Stores metadata for group chat rooms and their shared persistent AI conversation history.

| Field | Type | Attributes | Description |
| :--- | :--- | :--- | :--- |
| `_id` | ObjectId | Primary Key | MongoDB auto-generated ID. |
| `name` | String | required, max:50 | Display name of the room. |
| `description` | String | max:500 | Optional details about the room's purpose. |
| `tags` | Array of Strings | lowercase | Categorization labels. |
| `maxUsers` | Number | min:2, max:100, default:20 | Capacity limit for the room. |
| `creatorId` | ObjectId | required, Ref: `User` | ID of the user who created the room. |
| `aiHistory` | Array of Objects | - | Contextual history for Gemini AI in this room. |
| `aiHistory[].role` | String | enum: ['user', 'model'] | Speaker identity in the AI context. |
| `aiHistory[].parts[].text`| String | - | Actual text content sent to or returned by the AI. |
| `createdAt` | Date | auto | Timestamp of room creation. |
| `updatedAt` | Date | auto | Timestamp of last room update. |

*Indexes:*
- `creatorId` (Ascending)
- `tags` (Ascending)

### 3. Messages (`messages` collection)
Stores individual chat messages sent within Group Rooms.

| Field | Type | Attributes | Description |
| :--- | :--- | :--- | :--- |
| `_id` | ObjectId | Primary Key | MongoDB auto-generated ID. |
| `roomId` | ObjectId | required, index, Ref: `Room` | ID of the room this message belongs to. |
| `userId` | String | required | ID of the sender (or 'system'/'ai'). |
| `username` | String | required | Sender's display name. |
| `content` | String | required | Primary text content of the message. |
| `isAI` | Boolean | default: false | Flag indicating if the message was generated by Gemini. |
| `triggeredBy` | String | default: null | If `isAI=true`, the user ID who invoked the AI. |
| `replyTo.id` | String | default: null | ID of the parent message this replies to. |
| `replyTo.username` | String | default: null | Username of the parent message author. |
| `replyTo.content` | String | default: null | Snippet of the parent message content. |
| `reactions` | Map of [String] | default: {} | Key-value mapping of Emojis -> Array of User IDs. |
| `createdAt` | Date | auto | Message dispatch timestamp. |
| `updatedAt` | Date | auto | Message edit/update timestamp. |

*Indexes:*
- Compound: `{ roomId: 1, createdAt: -1 }` (Optimizes fetching recent messages for a room).

### 4. Conversations (`conversations` collection)
Stores entire histories of private 1-on-1 chats a user has with the AI.

| Field | Type | Attributes | Description |
| :--- | :--- | :--- | :--- |
| `_id` | ObjectId | Primary Key | MongoDB auto-generated ID. |
| `userId` | ObjectId | required, index, Ref: `User` | The owner of the conversation. |
| `title` | String | required, max:100 | Generated title for the chat thread. |
| `messages` | Array of Objects | - | The chronological sequence of messages. |
| `messages[].role` | String | enum: ['user', 'assistant'] | Who sent the message. |
| `messages[].content`| String | required | Markdown body of the message. |
| `messages[].timestamp`| Date | default: Date.now | When this specific message was recorded. |
| `createdAt` | Date | auto | Conversation creation time. |
| `updatedAt` | Date | auto | Last time a message was appended. |

*Indexes:*
- Compound: `{ userId: 1, updatedAt: -1 }` (Optimizes fetching a user's recent chats).

### 5. Refresh Tokens (`refreshtokens` collection)
Manages long-lived authentication tokens mapped to specific users.

| Field | Type | Attributes | Description |
| :--- | :--- | :--- | :--- |
| `_id` | ObjectId | Primary Key | MongoDB auto-generated ID. |
| `token` | String | required, unique, index | The secure random refresh token string. |
| `userId` | ObjectId | required, index, Ref: `User`| The user to whom this token belongs. |
| `expiresAt` | Date | required | Expiration timestamp. |
| `createdAt` | Date | auto | Issuance timestamp. |
| `updatedAt` | Date | auto | Last update timestamp. |

*Indexes:*
- TTL Index: `{ expiresAt: 1 }` with `expireAfterSeconds: 0`. MongoDB automatically deletes documents when `expiresAt` is reached in real-time.
